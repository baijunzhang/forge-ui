# AI 编程助手前端设计 Prompt

> 用途：喂给 AI，让其设计并实现一个 Claude Code / Codex 风格的 AI 编程助手前端界面，嵌入我们的 AI 软件。

---

## 角色
你是一名资深全栈工程师 + 产品设计师，擅长构建 AI 编程助手类产品的前端界面
（参考 Claude Code / Cursor / GitHub Copilot Workspace / OpenAI Codex 的交互范式）。

## 目标
为我们的 AI 软件设计并实现一个「AI 编程助手」前端界面（Web 端），
让用户能通过对话驱动 AI 阅读、修改、提交代码，并对每一处改动有完全的可视化掌控权。

## 必须包含的核心功能

### 1. 对话区（Chat Panel）
- 用户与 AI 的多轮对话流，支持 Markdown 渲染、代码块语法高亮
- 区分用户消息 / AI 消息 / 系统提示（工具调用、状态变更）
- 流式输出（逐字打字机效果），支持「停止生成」
- AI 调用工具时显示可折叠的工具调用卡片（如：读取文件、搜索、运行命令）

### 2. 代码改动可视化（核心，重点设计）
- Diff 视图：以 git diff 风格展示 AI 的修改
  - 新增行：绿色背景 + "+" 标记
  - 删除行：红色背景 + "-" 标记
  - 修改行：上下对照（删除旧行 + 新增新行）
  - 未改动的上下文行：常规颜色，可点击展开/折叠
- 支持「并排对照（split view）」和「行内对照（inline view）」两种模式切换
- 每个文件的改动单独成卡片，显示文件路径、+N/-M 行数统计
- 行号高亮，跳转定位

### 3. 改动审批（Accept / Reject）
- 每处改动可单独 Accept 或 Reject（hunk 级别粒度）
- 也支持整个文件 / 整批改动一键 Accept All / Reject All
- 提供「Accept Edits」模式开关：开启后 AI 的后续改动自动应用，无需逐个确认
- 被拒绝的改动给 AI 反馈，让它重新生成

### 4. 文件操作
- 文件树侧边栏：展示项目结构，标记有未提交改动的文件（圆点/颜色）
- 文件查看器：点击文件查看完整内容，带语法高亮
- 提交（Commit）：选择要提交的文件、填写 commit message、确认提交
- 支持新建文件、删除文件、重命名

### 5. 状态与权限
- 顶部状态栏：当前模型、当前分支、运行状态（思考中 / 等待审批 / 空闲）
- 权限模式：手动审批 / 自动接受编辑 / 计划模式（只读规划，不改代码）
- 操作前对不可逆动作（删除、提交、运行命令）二次确认

## 交互细节
- 键盘快捷键：Accept (⌘+Enter)、Reject (Esc)、切换 diff 视图等
- 改动应用时有平滑动画过渡，不要生硬跳变
- 空状态、加载态、错误态都要设计
- 响应式：桌面端为主，但要适配窄屏折叠侧边栏

## 输出要求
1. 先给出整体信息架构和组件树（不写代码，先对齐设计）
2. 待我确认后，再分模块输出可运行的前端代码
3. 每个组件附简短说明：职责、props、关键交互
4. 标注哪些地方需要后端接口，以及接口的数据结构（TypeScript 类型）

---

## 使用建议
1. **方括号 `[ ]` 的地方一定要填**（技术栈、配色、状态管理），否则 AI 会自行选择，可能与现有项目不一致。
2. **"先对齐设计、再写代码"很重要**：分两步走避免一次生成过多代码、来不及审查。
3. 若某功能是重中之重（如 diff 可视化），在 prompt 开头补一句强调，AI 会据此分配精力。
