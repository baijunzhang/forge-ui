### A3. 自动滚动应尊重用户(流式时别强拽到底)
- 现象:AI 流式输出时,聊天区强制把用户拉回底部,想往上翻看历史被拽下去。
- 正确行为(标准聊天):用户在底部才自动跟随;一旦手动上滚就停止跟随;滚回底部再恢复。
- 纯前端。指令:
  ```text
  Bug: while the AI is streaming/running, the chat area force-scrolls me back to the bottom,
  so I can't scroll up to read earlier messages — it drags me down to where it's generating.

  Fix (frontend-only, standard chat behavior):
  1. Only auto-scroll to bottom if the user is ALREADY at (or near) the bottom.
  2. As soon as the user scrolls UP during streaming, STOP auto-scrolling and let them read
     freely — do not yank them back down.
  3. Resume auto-follow only when the user scrolls back to the bottom themselves.
     (Optional: show a small "Jump to latest" button while they're scrolled up.)

  Impladvice: track a "stickToBottom" flag; set it false on manual scroll-up, true when the
  user is within ~40px of the bottom. Gate the auto-scroll on that flag.
  Verify by scrolling up while the AI is generating — the view must stay where I put it.
  ```
