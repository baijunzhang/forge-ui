### A2. 写入被拦(权限)——非 review 回归

### A3. 自动滚动应尊重用户(流式时别强拽到底)
- 现象:AI 流式输出时,聊天区强制把用户拉回底部,想往上翻看历史被拽下去。
- 正确行为(标准聊天):用户在底部才自动跟随;一旦手动上滚就停止跟随;滚回底部再恢复。
- 纯前端。指令:
  ```text
  Bug: while the AI is streaming/running, the chat area force-scrolls me back to the bottom,
  so I can't scroll up to read earlier messages — it drags me down to where it's generating.

  Fix (frontend-only, standard chat behavior):
  1. Only auto-scroll to bottom if the user is ALREADY at (or near) the bottom.
  2. As soon as the user scrolls UP during streaming, STOP auto-scrolling and let them read
     freely — do not yank them back down.
  3. Resume auto-follow only when the user scrolls back to the bottom themselves.
     (Optional: show a small "Jump to latest" button while they're scrolled up.)

  Impladvice: track a "stickToBottom" flag; set it false on manual scroll-up, true when the
  user is within ~40px of the bottom. Gate the auto-scroll on that flag.
  Verify by scrolling up while the AI is generating — the view must stay where I put it.
  ```

#### STEP P1 — 任务时间线（Read → Plan → Edit → Test → Summary）
```text
STEP P1 — Task timeline. CSS/frontend-only, keep all existing IDs/classes/bridge
handlers intact.

Placement: a single-row, compact progress strip at the top of the AI's current turn
(above its message bubble), not a separate panel.

Stages (derive from existing activity events per P0's findings):
  Read (file reads) → Plan (reasoning/planning text) → Edit (write/edit calls) →
  Test (bash/test runs) → Summary (final answer text)

Visual spec (match existing neutral theme — do not introduce new accent colors):
- Layout: horizontal row of 5 steps, each = [dot] label, connected by a thin 1px line.
  Dark theme line: rgba(255,255,255,.12). Light theme line: rgba(0,0,0,.12).
- Completed step: dot = 14px filled circle in the primary text color
  (dark #e8e8e8 / light #1a1a1a), with a small monochrome checkmark glyph inside
  (use currentColor, no colored icon). Label text: muted color (#a0a0a0 dark / #666 light).
- Current step: dot = 14px filled circle, same primary color, with a subtle pulse
  animation (opacity 0.6 → 1 → 0.6 over 1.4s, ease-in-out, infinite). Label text:
  primary color (#e8e8e8 dark / #1a1a1a light), font-weight 500 (not 700 — project
  uses only 400/500).
- Upcoming step: dot = 14px outline circle only (1px border, muted color, transparent
  fill). Label text: muted/faded color (#6e6e6e dark / #999 light).
- Connecting line between two completed/current steps: primary-ish (rgba at 40%
  opacity); between current and upcoming: the default faint 12% line.
- Typography: labels 12px, letter-spacing .04em, same treatment as existing all-caps
  section labels (SESSIONS / STUDIO). Font: Inter, weight 400/500 only.
- Spacing: 8px gap between dot and label, 20px gap between step groups. Row padding
  12px vertical, sits in its own row with 8px margin below before the message text.
- Do NOT use color (green/red/blue) for step states — reserve color exclusively for
  diff add/remove lines elsewhere in the app. This timeline stays monochrome.

Verify live: trigger a real multi-step task and confirm the strip updates step-by-step
in both light and dark themes. Report before/after screenshots description.
```

#### STEP P6 — 跨文件夹访问权限弹窗（好看的那种）
```text

(do not fake it with a client-side-only mock). Frontend styling should match the
existing neutral Codex/Claude theme exactly — reuse existing CSS variables for colors,

  with a faint neutral background (rgba(255,255,255,.06) dark / rgba(0,0,0,.04) light).

  weight, not alarm color.
- Title: "Allow access to this folder?" — 15px, weight 500, primary text color.
- Body: 13px, secondary color (#a0a0a0 dark / #666 light), line-height 1.6. Show the

  rgba(0,0,0,.04) light, border-radius 4px, padding 2px 6px, 12.5px size, truncate

  remember this folder as approved for the rest of the session (used to avoid
  re-prompting for the same folder).
- On Deny: cancel that specific operation only, tell the AI it was denied (so it can
  explain to the user in its own reply why it couldn't complete that step), and do NOT
  crash or silently hang.

Verify live: actually get the AI to attempt an out-of-scope path and confirm the real
dialog appears (not a mock), both Allow and Deny paths work, and the "allow for rest of
session" checkbox is respected on a second attempt. Report the exact event/guard you
wired this to from P5.
```
