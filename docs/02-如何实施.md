# 如何实施 — 一步步把 Forge AI IDE 做出来

> 给"未来在公司电脑上的自己":照着这份做就行,不用记。
> 核心原则:**令牌文件先立好别动 → 发总纲 → STEP 一次发一个,跑通再发下一个。**

---

## 总流程

```
① 建项目 + 放好 globals.css / tailwind.config.ts   ← 你手动,只做一次
② 发"开场白"(总纲) → AI 回 "Ready"
③ 依次发 STEP 1 → STEP 10,每步看一眼再发下一步
```

---

## ① 建项目 + 放令牌文件(你手动)

令牌文件是"宪法",先立好,**不要让 AI 生成或重写**,否则会被改乱。

```bash
# 1. 建 Next.js + TS + Tailwind 脚手架
npx create-next-app@latest forge --ts --tailwind --app
cd forge

# 2. 用本仓库现成的两个文件覆盖默认的:
#    app/globals.css      ← 覆盖
#    tailwind.config.ts   ← 覆盖
#    (直接从 forge-ui 仓库复制过去)

# 3. 跑起来确认能启动
npm run dev
```

> 两个现成文件:
> - `app/globals.css` — 所有颜色/字体/间距令牌 + 轻盈动效工具类
> - `tailwind.config.ts` — 令牌映射成语义类名(bg-surface / text-secondary / accent ...)

---

## ② 发开场白(总纲)

把下面这一整段复制发给 AI。它应该回 "Ready" 并等待。
**如果它没回 Ready 就直接开始写一堆代码 → 打断,重发。**

```text
You are a senior frontend engineer building an AI IDE (like Claude / Cursor),
using React + TypeScript + Tailwind + shadcn/ui.

We will build it INCREMENTALLY, one small step at a time.

Rules you MUST follow:
1. Do ONLY the current step. Do not build ahead.
2. After each step, the app must compile and run with no errors.
3. Keep it simple. Prefer fewer, cleaner components over many clever ones.
4. Reuse existing components and styles. Do not reinvent.
5. After writing code, self-review: list what you changed, and any risk/bug you see.
6. Use placeholder/mock data first. Wire real logic only when a step asks for it.
7. If a step is ambiguous, ask ONE clarifying question before coding.

Design tokens are ALREADY set up in app/globals.css and tailwind.config.ts.
DO NOT generate or modify those two files. Just use the semantic classes:
bg-surface / bg-elevated / text-secondary / accent / border-subtle, etc.
- Never hardcode hex colors or font sizes.
- No pure black or pure white. No font-weight above 600.
- 8px spacing grid; use the defined radius/shadow tokens.

Keep the UI LIGHTWEIGHT, not heavy:
- Separate panels with 1px subtle borders + whitespace, not shadows/thick fills.
- Icon buttons are transparent; show bg-hover only on hover.
- Expand/collapse (toolbar, tool-call cards, panels) uses a height + fade
  animation (.collapsible / animation: slide-down). No abrupt jumps.
- Collapse secondary panels (right inspector, bottom terminal) BY DEFAULT.
- Animations: 120-260ms, ease cubic-bezier(0.2,0,0,1), animate only
  opacity/transform. No bounce, no large motion, no looping (except loaders).

Tech: dark mode first, clean and minimal (Linear / Claude aesthetic).

Reply "Ready" and wait. I will give you ONE step at a time.
```

## ③ 依次发 STEP(一次一个!)

**不要一次贴十个**——一次贴十个就退回到"一次做太多 → 又丑又 bug"的老问题了。

每步的节奏:
```
发 STEP N
  → AI 写代码 + 自检
  → 你在浏览器跑一下看一眼 (npm run dev)
  → 没问题 → 发 STEP N+1
  → 有 bug  → 回它:"这里有问题 [描述],先修好再继续",不要往下发
```

STEP 1–10 的具体内容见 `AI-IDE-Stepwise-Prompt.md`。简表:

| STEP | 做什么 |
|------|--------|
| 1 | 布局骨架(左栏/中间/右栏 + 顶栏)|
| 2 | 对话区(静态 mock 消息)|
| 3 | 输入框(发送/Enter)|
| 4 | 流式输出(打字机效果)|
| 5 | 工具调用卡片(可折叠)|
| 6 | 文件树(改动标记)|
| 7 | **Diff 视图(红绿改动)★核心** |
| 8 | **Accept / Reject 审批** |
| 9 | 提交(选文件 + message)|
| 10 | 打磨(空状态/loading/对比度,不加新功能)|

> STEP 7 是灵魂,可以在那步多花时间让它打磨干净再继续。

## 速查表

| 东西 | 谁做 | 怎么做 |
|------|------|--------|
| globals.css / tailwind.config.ts | **你,手动** | 复制现成的覆盖进项目,别让 AI 改 |
| 开场白(总纲) | 发给 AI | 复制本文件 ② 那段,等它回 Ready |
| STEP 1–10 | 发给 AI | 一次一个,跑通再发下一个 |

---

## 常见翻车点

- ❌ 一次把十个 STEP 全贴给 AI → ✅ 一次一个
- ❌ 让 AI 重新生成 globals.css / tailwind → ✅ 用现成的,明确禁止它改
- ❌ 没跑就发下一步,bug 越堆越多 → ✅ 每步浏览器看一眼
- ❌ AI 没回 Ready 就狂写代码 → ✅ 打断重发总纲
- ❌ 某步出 bug 还硬往下走 → ✅ 停下,先修好这步
