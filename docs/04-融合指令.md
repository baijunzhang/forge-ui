# 融合指令 — 把 Forge AI IDE 界面接进现有软件

> 给"未来在公司电脑上的自己":要让 AI(Codex / Claude Code 等)把这套界面融合进公司软件时,直接复制下面的指令,并照"注意事项"先确认两件事。

---

## 一、先确认两件事(不然 AI 容易卡住)

**1. AI 能不能拿到代码?**
- AI 能联网 / 能 `git clone` → 直接给仓库链接即可:
  `https://github.com/baijunzhang/forge-ui`
- AI 不能联网 → 先自己把仓库 clone 到公司项目能访问的位置,再让 AI 在那个文件夹里操作。

**2. 要替换哪个页面?**
- 明确告诉 AI:替换掉现有软件的哪个文件 / 哪个页面。不说清楚它不知道往哪放。

---

## 二、可直接复制的融合指令

```text
请把这个仓库的 AI IDE 界面融合进我们现有软件的某个页面:
https://github.com/baijunzhang/forge-ui

背景:
- 这是一个全屏独立页,纯 HTML + CSS + 原生 JavaScript,不用 npm / 框架 / 构建工具。
- 共 4 个文件:
  index.html  —— 页面结构
  styles.css  —— 设计令牌(颜色/字体/间距/动效),不要改
  app.css     —— 组件样式
  app.js      —— 交互逻辑(已包在 IIFE 里)

要求:
1. 把它整合进【这里写要替换的页面/文件名】,替换掉旧界面。
2. 保留我们软件自己的功能不受影响。
3. 整个 IDE 已包在 .forge-ide 作用域 + IIFE 里,不要破坏这个隔离,
   避免污染我们软件其他页面的 CSS / 全局 JS。
4. app.js 里的 mock 数据(TREE / MOCK_DIFF / streamReply)是接入点,
   先保留运行,之后再对接我们自己的 AI / 本地读写逻辑。
5. 本地文件读写用的是浏览器原生 File System Access API(openLocalFolder /
   readFile / writeFile),已实现真功能,保留即可。

做完后告诉我:你改了哪些文件、怎么验证界面能打开。
```

> 把【】里的内容换成你实际要替换的页面名。

---

## 三、融合后怎么验证

1. 用 Chrome / Edge 打开那个页面(File System Access API 需要 Chromium 内核)。
2. 检查:三栏布局正常、深浅色切换正常、diff 红绿正常、标签可切换。
3. 点左上角"连接本地文件夹" → 应弹出系统选择框 → 选中后显示真实本地文件。
4. 确认我们软件的其他页面没被影响(样式没串、控制台无报错)。

---

## 四、常见问题

- **AI 想改成 React / 装 npm** → 打断它:"纯 HTML,不要框架,不要 npm。"
- **AI 想改 styles.css 的令牌** → 阻止:"styles.css 是设计规范,不要动。"
- **本地文件夹按钮没反应** → 多半是用了 Safari 或 file:// 打开;换 Chrome/Edge,或通过本地服务器打开。
- **样式串到别的页面** → 检查是否所有规则都在 .forge-ide 作用域内、JS 是否在 IIFE 内。

---

## 五、提醒:不要只说一句话

❌ 只说"把这个 index.html 融合进来" → AI 信息不足,容易做错。
✅ 用上面第二节那段完整指令,并填好要替换的页面名。
